# Side-by-Side Grammatical Corrections

### Q1: Tell me about yourself

> ❌ **Original:** "I am backend developer and I working with Python for five years now."
> ✅ **Corrected:** "I am **a** backend developer, and I **have been working** with Python for five years now."
> 📌 **Rule:** Use `a/an` before job titles (`a backend developer`). Use the Present Perfect Continuous (`have been + -ing`) for actions started in the past that continue into the present.

---

> ❌ **Original:** "In my last job, I make a microservice that process millions of webhook events every day."
> ✅ **Corrected:** "In my last job, I **built** a microservice that **processes** millions of webhook events every day."
> 📌 **Rule:** Use the past tense (`built/made`) for completed past achievements. Use the third-person singular `-s` ending (`processes`) for singular nouns/pronouns in the present simple.

---

### Q2: Database Scaling Challenges

> ❌ **Original:** "When we have big traffic spikes, PostgreSQL was get very slow because too many connections."
> ✅ **Corrected:** "When we **had** big traffic spikes, PostgreSQL **became** very slow because **of** too many connections."
> 📌 **Rule:** Maintain consistent past tense when describing past incident responses (`had/became`). Use `because of + noun phrase` (`because of too many connections`) or `because + full clause`.

---

> ❌ **Original:** "If the data is read-only or session data, we put in Redis cache so the database don't crash."
> ✅ **Corrected:** "If the data is read-only or session data, we put **it** in **a** Redis cache so the database **doesn't** crash."
> 📌 **Rule:** Transitive verbs like `put` require an explicit direct object (`put it`). Singular nouns (`the database`) take `doesn't` instead of `don't`.

---

### Q3: Remote International Role Transition

> ❌ **Original:** "Because I want work with global engineering standards and high-scale distributed systems."
> ✅ **Corrected:** "I **want to** work with global engineering standards and high-scale distributed systems."
> 📌 **Rule:** The verb `want` is followed by an infinitive with `to` (`want to work`). Avoid starting formal interview sentences with standalone `Because`.

---

> ❌ **Original:** "I have discipline for async work and I document everything in markdown so the timezone difference is not problem for me."
> ✅ **Corrected:** "I have **strong** discipline for async work and document everything in markdown, so the timezone difference is not **a** problem for me."
> 📌 **Rule:** Singular countable nouns like `problem` must be preceded by an article (`a problem`).
